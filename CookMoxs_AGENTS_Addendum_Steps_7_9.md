This file is a merge/addendum instruction set for the existing `AGENTS.md`.

It does not replace the repo’s current AGENTS rules.
It adds the rules needed for the Steps 7–9 visual production track.

Use this addendum together with:

## Add these rules to AGENTS behavior for Steps 7–9

### 1. Treat Steps 1–6 as accepted baseline
Do not casually reopen:
- button system
- surface/card system
- bottom nav direction
- dark-shell/light-card polarity
- accepted Settings/Recipe structure
except where direct regressions from Steps 7–9 require tiny corrective changes.

### 2. Autumn remains the base world
For asset integration:
- Autumn is default/base
- other seasons are tonal/runtime variants of the same product
- no season may behave like a separate app

### 3. Vivaldi remains hidden
Never literalize Vivaldi.
Keep it hidden in:
- rhythm
- pacing
- density
- tonal movement
- motion phrasing
not as symbols, decorative notation, or thematic gimmicks.

### 4. Cook Mode remains sacred
For all Steps 7–9:
- function beats atmosphere
- readability beats motion
- neutral base beats scenic temptation
- overlays must isolate cleanly
- HUD and sticky structures must remain stable

### 5. Asset integration is not asset invention
Step 7 is integration, not generation.
Do not invent new backgrounds or reinterpret approved assets unless explicitly asked.

### 6. Motion must stay restrained
For Step 8:
- no flashy motion
- no game-like motion
- no bounce/spectacle
- support reduced motion
- motion must support clarity, not perform personality

### 7. Release hardening is regression-first
For Step 9:
- if polish conflicts with stability, preserve stability
- no “one more redesign”
- tiny fixes only
- regression control before ambition

### 8. Prefer structural fixes over decorative patching
Do not solve Steps 7–9 problems with:
- extra shadows
- extra badges
- more blur
- extra ornamental effects
Prefer:
- safer stacking
- better mapping
- better padding
- better containment
- better state/motion discipline

### 9. Continue sequentially unless blocked
For the Steps 7–9 run:
- execute Step 7
- then Step 8
- then Step 9
continue automatically unless a hard blocker or broken build prevents safe continuation.

### 10. Report per step
For each step, report:
1. completed scope
2. files changed
3. key implementation decisions
4. checks run
5. could-not-verify items
6. exact screenshots/review flows required

### 11. Do not lose approved runtime corrections
Do not accidentally regress:
- household mobile stack
- recipe meta order
- ingredient right rail
- sticky CTA clearance
- product-data mobile safety
- cook-mode overlay isolation
- selected-state clarity
- settings containment work

### 12. Final review must be practical
By the end of Step 9, Codex must leave the branch in a state where:
- a human can run a clean screenshot review
- remaining uncertainty is explicit
- merge/release readiness can be judged honestly
